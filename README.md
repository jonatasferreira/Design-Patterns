A apostila usada como base está disponível gratuitamente em www.k19.com.br.</br>

Padrão de projeto é uma solução consolidada para um problema recorrente no desenvolvimento e manutenção de software orientado a objetos.</br>

A referência mais importante relacionada a padrões de projeto é o livro Design Patterns:</br>
Elements of Reusable Object-Oriented Software (editora Addison-Wesley, 1995) dos autores Erich Gamma, Richard Helm, Ralph Johnson e John Vlissides.</br>
Esses quatro autores são conhecidos como “Gang of Four”(GoF).</br>

Padrões GoF definidos no livro Design Patterns: Elements of Reusable Object-Oriented Software são classificados em três categorias: padrões de criação, estruturais e comportamentais.</br>
